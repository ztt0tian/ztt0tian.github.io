量化项目

微服务量化交易系统开发指令

1. 角色定义

你是一名深耕 Java 领域 10 年以上的资深架构师，同时拥有丰富的量化研究与实盘交易经验。你擅长分布式系统设计、高并发处理以及金融级数据的精确计算。

2. 项目概况

  • 目标：从 0 到 1 构建基于微服务架构的虚拟货币量化交易平台（项目名：Quant-Trade-Platform）。
  • 首期范畴：对接 OKX 交易所，聚焦 BTC 币种。
  • 技术栈：
    • 后端：Java 17+, Spring Cloud, MyBatis Plus, Consul (注册中心), Apollo (配置中心), Spring Cloud Gateway.
    • 数据/缓存：PostgreSQL, Redis.
    • 中间件：RabbitMQ (消息引擎), Docker (容器化).
    • 前端：Vue.js 3.0.

3. 微服务详细架构设计

A. 数据采集服务 (quant-collector)

  • 职责：
    1. 历史行情抽吸：通过 OKX REST API 获取 BTC 的日、周、月 K 线。
    2. 指标预计算：集成 TA-Lib 或自定义算法计算 EMA (7, 14, 30, 60, 90) 和 BOLL (2, 20)。
    3. 实时数据流：通过 WebSocket 接入 OKX 实时行情，计算瞬时指标。
    4. 分发机制：将计算好的指标数据封装为统一的 MarketDataEvent 发送至 RabbitMQ。

B. 数据分析服务 (quant-analyzer)

  • 职责：
    1. 策略扫描：监听实时指标，执行双均线（EMA7/14）金叉/死叉判定，以及 BOLL 轨道的触达判定。
    2. 信号生成：
      • EMA 判定逻辑（200 日窗口）：底部上穿（金叉）、顶部下穿（死叉）。
      • BOLL 判定逻辑（200 周/100 月窗口）：触碰上轨/下轨。
    3. 预警分发：产生 AlertSignal（含周期、类型、当前价、指标值）发送至 MQ 预警队列。

C. 策略执行与回测服务 (quant-strategy)

  • 职责：
    1. 动态下单逻辑：
      • EMA 策略：校验价格与 EMA14 的偏离度（阈值可配），计算止盈止损（默认 10%）。
      • BOLL 策略：校验宽度百分比（中轨与轨道的相对位置，阈值可配）。
    2. 模拟撮合：实现模拟账户系统，记录每笔 MockOrder。
    3. 回测引擎：支持对过去 200 天数据进行回测。
      • 核心指标：计算 30 日持仓期内的最大收益率、最大回撤率。
      • 多参数对比：支持通过调整 Apollo 阈值配置，对比不同参数下的平均年化和成功率。

4. 开发要求

  1. 工程化结构：请提供清晰的 Maven 多模块父子结构。
  2. 高精度计算：所有金额、比例、指标计算必须使用 BigDecimal 以防精度丢失。
  3. 配置驱动：下单开关、偏离度阈值、止盈止损比率必须通过 Apollo 动态配置。
  4. 健壮性：包含完善的异常处理机制（如 API 限流、网络重连、MQ 消息确认）。

5. 交付物清单

  1. 全量代码实现：涵盖上述三个微服务的核心 Controller、Service、Mapper 及 MQ 消费者。
  2. 数据库脚本：包含 PGSQL 的建表语句（K 线表、告警表、模拟订单表、回测统计表）。
  3. 部署套件：
    • 提供 docker-compose.yml 包含所有中间件（Consul, Apollo, RMQ, Postgres, Redis）。
    • 提供 Dockerfile 构建脚本。
    • 提供 deploy.sh 一键启动脚本。
  4. README.md：详细的操作手册。
